Installation

  • Add the spike_tb directory to the matlab path
  • The spike toolbox requires some utilitiy functions in ../utilities. Add this directory to the matlab path.
  • The PCI-AER stimulation functions are developmental, but require a built copy of stimmon on the path. The easiest option is to add a symbolic link to a built version. stimmon is available in the alavlsi repository, under SW/c/poisson.

Help and basic usage guidelines

  • The Spike Toolbox currently has no formal documentation. However, each function has syntax and usage help available by typing 'help (function name)' or 'helpwin (function name)' at the matlab command line.
  • Type helpwin spike_tb in matlab to get a list of toolbox functions with descriptions.
  • Type helpwin STFormats for some basic information on the spike train objects, their representation and usage.
  • Type helpwin STAddrSpecInfo for information on how to specify on-chip neuron addresses to the toolbox.
  • Errors are reported on the command line with three asterisks and the name of the function raising the error prepended. For example
    *** STAddrSpecCompare: Invalid specification
    When a function raises an error, it does not guarantee to return valid output, or indeed any output at all.
  • Warnings are reported on the command line with three dashes and the name of the function raising the warning prepended. For example
    --- STIsValidAddress: Extra arguments ignored
    When a function raises a warning, it guarantees to still return valid output as definied by the function's help text.

Getting started

  • Use STCreate to create a new spike train definition.
  • Instantiate the definition with STInstantiate. (This converts an abstract definition into an actual train of spikes with some duration.)
  • Map the spike train to a neuron and synapse address with STMap.
  • Multiplex several spike trains together with STMultiplex.
  • Stimulate your neurons with STStimulate.
  • View the results of your stimulus (or the stimulus itself) with STPlotRaster.
  • Use STOptions to retrieve and change the spike toolbox options.
  • STDescribe will print a human-friendly summary of most Spike Toolbox variables. This is useful particularly when defining addressing modes, and to easily verify the toolbox options.
